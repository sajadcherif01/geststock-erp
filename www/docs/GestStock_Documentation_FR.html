1. Objectif de l'application

GestStock ERP V4 centralise la gestion commerciale et logistique d'une activite de tapis/moquette : articles, clients, fournisseurs, sites, achats, ventes, transferts, inventaires, comptes et analyses.

Principe general : les donnees de reference alimentent les operations. Les operations modifient le stock et les comptes. Les tableaux de bord et rapports exploitent ces donnees pour donner une vision de gestion.

4. Schema des donnees

Reference

Articles

Nom, type tapis/moquette, prix par defaut, dimensions utilisees dans les operations.

Reference

Clients / Fournisseurs

Nom, ville, solde initial et rattachement aux operations commerciales.

Reference

Sites

Magasins ou depots permettant de localiser le stock.

Prix

Prix clients / fournisseurs

Prix au m2 par article et par tiers pour calculer automatiquement les montants.

Mouvements

Achats / Ventes / Transferts

Quantite, article, couleur, dimensions, site, prix et total.

Comptes

Paiements

Mode, montant, date, echeance, statut paye/impaye et deduction immediate ou differee.

7. Hebergement GitHub Pages

L'application etant un fichier HTML autonome, elle peut etre publiee directement avec GitHub Pages. Le fichier doit s'appeler index.html a la racine du depot.

  • Depot recommande : geststock-erp
  • Branche : main
  • Source GitHub Pages : Deploy from a branch / root
  • URL attendue : https://sajadcherif01.github.io/geststock-erp/

8. Points importants

  • Les donnees sont stockees dans le navigateur avec localStorage.
  • Un export de sauvegarde est necessaire avant migration ou changement de poste.
  • L'import Excel depend du chargement de la bibliotheque XLSX depuis CDN si elle n'est pas deja disponible.
  • GitHub Pages heberge l'interface, mais ne transforme pas l'application en base de donnees multi-utilisateur.
